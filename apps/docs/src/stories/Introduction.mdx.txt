# Prismia Design System

Bem-vindo à documentação do Prismia Design System.

## O que é o Prismia?

O Prismia é um Design System completo e enterprise-ready, desenvolvido para garantir consistência visual, acessibilidade e experiência de desenvolvimento otimizada em todos os projetos.

## Princípios

### Consistência
Todos os componentes seguem rigorosamente os tokens de design, garantindo uniformidade visual em toda a aplicação.

### Acessibilidade
Componentes desenvolvidos seguindo as diretrizes WCAG 2.1 Level AA, com suporte a navegação por teclado, leitores de tela e contraste adequado.

### Developer Experience
APIs intuitivas, TypeScript first, documentação completa e exemplos práticos.

## Arquitetura de Tokens

O Prismia utiliza uma arquitetura de tokens em 3 camadas:

### Layer 1: BASE
Tokens físicos - valores puros de cor, espaçamento, etc.

```css
--base-brand-lavender: #D8CCE8;
--base-contrast-graphite: #3C3F45;
```

### Layer 2: SEMANTIC
Mapeamento funcional dos base tokens.

```css
--semantic-primary-default: var(--base-brand-lavender);
--semantic-text-primary: var(--base-contrast-graphite);
```

### Layer 3: COMPONENT
Tokens específicos de componentes.

```css
--component-button-primary-bg: var(--semantic-primary-default);
--component-button-primary-text: var(--semantic-text-primary);
```

## Tokens Usados no Button

O componente Button consome os seguintes tokens:

### Cores
- `--component-button-primary-bg`: Background do botão primário
- `--component-button-primary-bg-hover`: Background hover primário
- `--component-button-primary-bg-active`: Background active primário
- `--component-button-primary-text`: Texto do botão primário
- `--component-button-secondary-bg`: Background do botão secundário
- `--component-button-secondary-border`: Borda do botão secundário
- `--semantic-hover-overlay`: Overlay de hover para ghost

### Espaçamento
- `--spacing-3`: Padding horizontal small (12px)
- `--spacing-4`: Padding horizontal medium (16px)
- `--spacing-6`: Padding horizontal large (24px)

### Radius
- `--radius-s`: Border radius small e medium (8px)
- `--radius-m`: Border radius large (12px)

### Typography
- `--font-weight-medium`: Peso da fonte (500)
- `--text-body-small`: Tamanho small (14px)
- `--text-body`: Tamanho medium (16px)

### Motion
- `--motion-duration-normal`: Duração de transição (200ms)
- `--motion-easing-default`: Easing padrão

## Acessibilidade

### Foco Visível
Todos os botões possuem indicador de foco visível para navegação por teclado:
- Outline de 2px com cor do focus ring
- Offset de 2px para melhor visualização

### Estados
- `default`: Estado padrão do botão
- `hover`: Feedback visual ao passar o mouse
- `active`: Feedback ao pressionar
- `disabled`: Estado desabilitado com opacidade reduzida
- `focus-visible`: Outline visível apenas na navegação por teclado

### Contraste
Todas as combinações de cores atendem WCAG 2.1 Level AA:
- Primary: Texto grafite (#3C3F45) sobre lavanda (#D8CCE8) - 4.8:1
- Secondary: Texto grafite sobre transparente com borda
- Ghost: Texto grafite sobre hover overlay

### ARIA
O componente Button utiliza semântica HTML nativa (`<button>`), garantindo:
- Acessibilidade automática via elementos nativos
- Suporte a leitores de tela sem configuração adicional
- Navegação por teclado (Enter/Space para ativar)

## Como Usar

### Instalação

#### Com Tailwind CSS
```bash
npm install @prismia/ui-tailwind @prismia/tokens
```

#### Com CSS Modules
```bash
npm install @prismia/ui-css @prismia/tokens
```

### Uso Básico

```tsx
import { Button } from '@prismia/ui-tailwind';

function App() {
  return (
    <Button variant="primary" size="md">
      Clique aqui
    </Button>
  );
}
```
